# POP-013: Aldaketa Kudeaketa Prozedura (Change Management)

**Helburua:** Informazioaren segurtasunean eragina izan dezaketen aldaketa guztiak modu kontrolatuan kudeatzea.
**Arduraduna:** IT Manager

## 1. Aldaketa Eskaera (RFC)
Aldaketa guztiak RFC (Request for Change) inprimaki baten bidez eskatu behar dira, honako hauek zehaztuz:
- Aldaketaren deskribapena
- Justifikazioa
- Inpaktuaren analisia
- Atzera egiteko plana (Rollback plan)

## 2. Ebaluazioa eta Onarpena
CAB (Change Advisory Board) taldeak aldaketa ebaluatuko du.
- **Aldaketa normalak:** Asteko bileran onartuak.
- **Aldaketa urgenteak:** CISO edo IT Manager-en berehalako onarpena behar dute.

## 3. Inplementazioa
- Garapen/Test ingurunean probatu lehenik.
- Produkzioan inplementatu baimendutako leihoan.

## 4. Berrikusketa
Aldaketa egin ondoren, funtzionamendua egiaztatu eta RFC itxi.

**Erregistroak:** Jira Change Management / RFC Tickets
